# 知识库管理弹窗改造计划

## 目标

将侧边栏底部的「知识库」按钮从切换全屏页面（KbManagePage）改为打开一个 Modal 弹窗，弹窗内用两个标签（上传文件 / 知识库列表）融合原本的上传和文件列表功能。

## 当前架构

```
Sidebar
  └─ onViewChange('kb') → App: appView='kb' → 渲染 KbManagePage（全屏）
  └─ onViewChange('chat') → App: appView='chat' → 渲染 ChatPanel

ChatPanel（在 chat 视图下）
  └─ 📎 上传文件 按钮 → 展开 upload-zone（简单拖拽上传，调用 uploadFilesBatch）
  └─ 📂 知识库列表 按钮 → 打开 KbListModal（公用/个人标签 + 搜索 + 删除）

KbListModal（Modal）
  └─ 公用知识库 / 个人知识库 标签
  └─ 搜索框
  └─ 文件列表（调用 listUploadedFiles API，支持删除）

KbManagePage（全屏页面）
  └─ 上传新文件 标签 → KbUploadTab（拖拽区 + 待上传列表 + 确认上传，用 store 的 pendingFiles/uploadProgress）
  └─ 已上传文件 标签 → KbFilesTab（store 的 uploadedFiles 列表 + 删除）
```

## 改造方案

### 1. 新建 KbManageModal.tsx

创建 `/frontend/src/components/KbManageModal.tsx`，作为 Ant Design Modal 组件，内含两个标签：

**左上角两标签**：
- 「上传文件」标签：复用 KbManagePage 的 KbUploadTab 逻辑（拖拽区 + 待上传列表 + 进度显示 + 确认上传）
- 「知识库列表」标签：复用 KbListModal 的文件列表逻辑（公用/个人标签 + 搜索 + 删除）

```tsx
<Modal title="知识库管理" open={open} onCancel={onClose} width={680} footer={null}>
  {/* 两个标签在左上角 */}
  <div style={{ display: 'flex', gap: 4, marginBottom: 20 }}>
    <button onClick={() => setTab('upload')}>上传文件</button>
    <button onClick={() => setTab('files')}>知识库列表</button>
  </div>
  
  {tab === 'upload' ? <UploadSection /> : <FilesSection />}
</Modal>
```

**UploadSection**：提取 KbManagePage 中 KbUploadTab 的逻辑，做成内嵌组件
- 拖拽/点击上传区
- 待上传文件列表（含文件类型图标、移除按钮）
- 确认上传按钮
- 上传进度显示

**FilesSection**：提取 KbListModal 中的文件列表逻辑
- 公用知识库 / 个人知识库 标签切换
- 搜索框
- 文件列表（文件类型图标、文件名、大小、日期、删除按钮）
- 空状态提示

### 2. 修改 Sidebar.tsx

- 接口增加 `onOpenKbManage: () => void` 属性
- 底部「知识库」按钮：移除 `currentView` 判断逻辑，改为调用 `onOpenKbManage()`
- 按钮始终显示为普通样式（不需要 currentView 的高亮状态）

### 3. 修改 App.tsx

- 新增状态：`const [kbManageOpen, setKbManageOpen] = useState(false)`
- Sidebar 的 `onViewChange` 替换为 `onOpenKbManage={() => setKbManageOpen(true)}`
- 渲染 `<KbManageModal open={kbManageOpen} onClose={() => setKbManageOpen(false)} />`
- 移除 `appView` 相关逻辑（或保留 KbManagePage 作为备用但不再从 Sidebar 触发）

### 4. KbManagePage 处理

保留 KbManagePage.tsx 文件不删除，但 Sidebar 不再切换到此页面。这样不影响已有代码，且 KbManagePage 的 KbUploadTab/KbFilesTab 组件逻辑可以继续复用。

## 文件修改清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `components/KbManageModal.tsx` | 新建 | 弹窗组件，内含上传和列表两个标签 |
| `components/Sidebar.tsx` | 修改 | 添加 `onOpenKbManage` prop，按钮改为打开弹窗 |
| `App.tsx` | 修改 | 新增弹窗状态，接入 KbManageModal |
| `App.css` | 修改 | 添加弹窗内标签样式（复用 kb-tab 样式或新增） |
| `components/chat.css` | 无需修改 | 样式复用现有 |

## 代码复用策略

- 上传功能：直接复制 KbManagePage 中 KbUploadTab 的 JSX 和逻辑，放在 Modal 的 body 内
- 文件列表：直接复制 KbListModal 中 tab + search + 文件列表的 JSX 和逻辑
- 样式：复用 App.css 中已有的 `.kb-upload-zone`, `.kb-pending-*`, `.kb-progress-*`, `.kb-tab` 等样式

## 验证步骤

1. TypeScript 检查：`cd frontend && npx tsc --noEmit`
2. 构建检查：`cd frontend && npx vite build`
3. 功能检查：
   - 点击侧边栏「知识库」→ 弹窗打开
   - 上传文件标签 → 拖拽/选择文件 → 确认上传 → 进度显示 → 完成
   - 知识库列表标签 → 公用/个人切换 → 搜索 → 删除文件
   - 关闭弹窗 → 回到对话页面（对话数据不受影响）
