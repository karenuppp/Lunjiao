# Lunjiao 部署与启动指南

> 部门智能问答系统 — 自然语言查数据、做分析、出报告/图表

---

## 项目结构概览

```
Lunjiao/
├── backend/                     # Python 后端 (FastAPI)
│   ├── app/
│   │   ├── main.py              # FastAPI 入口 (端口 8000)
│   │   ├── config.py            # 配置 (加载 .env)
│   │   ├── api/                 # API 路由: chat, upload, data_sources, history
│   │   ├── agent/               # LangGraph Agent
│   │   └── mcp_servers/         # MCP 独立服务器
│   │       ├── db_server.py     # MySQL 只读查询 (端口 8024)
│   │       └── upload_server.py # 上传文件查询 (端口 8025)
│   ├── requirements.txt
│   ├── .env                     # 环境变量 (实际配置, 已 gitignore)
│   ├── .env.example             # 环境变量模板
│   └── venv/                    # Python 虚拟环境 (Python 3.11)
├── frontend/                    # React 前端 (Vite)
│   ├── package.json
│   ├── vite.config.ts           # Vite 配置 (端口 5173, 代理 /api -> :8000)
│   └── src/
└── docs/
    └── 部署启动指南.md           # ← 本文档
```

## 二、后端部署

### 2.1 环境配置

```bash
cd Lunjiao/backend

# 复制环境变量模板并修改
cp .env.example .env
```

编辑 `.env`，参考以下配置：

```env
# LLM 模型 (本地 LM Studio)
OPENAI_API_KEY=***                          # LM Studio 不校验 key
OPENAI_BASE_URL=http://localhost:1234/v1
MODEL_NAME=qwen3.6-35B-A3B-apex

# MySQL 数据库
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的密码
DB_NAME=lunjiao

# RAG-Anything 检索
RAG_API_BASE=http://localhost:8023
RAG_API_KEY=ragflo...0006

# MCP 数据库服务器
MCP_SERVER_BASE=http://localhost:8024
```

### 2.2 安装依赖

```bash
cd Lunjiao/backend

# 激活虚拟环境
source venv/bin/activate

# 如未创建虚拟环境:
# python3 -m venv venv
# source venv/bin/activate

# 安装依赖 (如需要清华镜像)
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

### 2.3 启动服务

**打开终端 1 — 主 API 服务 (端口 8000):**

```bash
cd Lunjiao/backend
source venv/bin/activate
uvicorn app.main:app --reload --port 8000
```

> 访问 http://localhost:8000/docs 查看 Swagger API 文档
> 访问 http://localhost:8000/api/health 检查健康状态

**终端 2 — MCP 数据库服务器 (端口 8024):**

```bash
cd Lunjiao/backend
source venv/bin/activate
python -m app.mcp_servers.db_server
```

> 提供 MySQL 只读查询能力 (安全的 SELECT 过滤)
> 健康检查: http://localhost:8024/health

**终端 3 — MCP 上传文件服务器 (端口 8025):**

```bash
cd Lunjiao/backend
source venv/bin/activate
python -m app.mcp_servers.upload_server
```

> 支持 CSV/XLSX 上传与自然语言查询
> 健康检查: http://localhost:8025/health

> **注意:** 如果不需要文件上传功能，可以跳过此服务。如果不需要数据库查询，可以跳过 MCP db_server。

## 三、前端部署

### 3.1 安装依赖

```bash
cd Lunjiao/frontend
npm install
```

### 3.2 启动开发服务器

```bash
cd Lunjiao/frontend
npm run dev
```

> 默认运行在 http://localhost:5173
> Vite 自动将 `/api/*` 请求代理到 `http://localhost:8000` (配置见 `vite.config.ts`)

### 3.3 生产构建

```bash
cd Lunjiao/frontend
npm run build
```

> 产物输出到 `frontend/dist/` 目录

## 四、快速启动 (一站式)

同时启动所有必要服务：

```bash
# 终端 1: 后端 API
cd Lunjiao/backend && source venv/bin/activate && uvicorn app.main:app --reload --port 8000

# 终端 2: MCP 数据库
cd Lunjiao/backend && source venv/bin/activate && python -m app.mcp_servers.db_server

# 终端 3: 前端
cd Lunjiao/frontend && npm run dev
```

然后访问 http://localhost:5173

## 五、外部依赖

以下服务需在 **启动项目前** 确认运行：

### 5.1 LM Studio (本地 LLM)

- 端口: 1234
- 模型: `qwen3.6-35B-A3B-apex` (与 `.env` 的 `MODEL_NAME` 一致)
- 启动后确认: `curl http://localhost:1234/v1/models`

### 5.2 MySQL

- 端口: 3306
- 数据库: `lunjiao` (需要提前创建)
- 连接测试: `mysql -u root -p -e "USE lunjiao; SHOW TABLES;"`

> **首次部署需初始化用户表：**
> ```bash
> cd Lunjiao/backend
> source venv/bin/activate
> python init_users.py
> ```
> 默认创建测试账号 `193699 / 193699`。可通过参数覆盖：
> ```bash
> python init_users.py --account admin --password admin123
> ```

### 5.3 RAG-Anything (文档检索)

- 端口: 8023
- 健康检查: `curl http://localhost:8023/health`

## 七、常见问题

**Q: 前端请求报 CORS 错误？**
后端已配置 CORS 允许 `localhost:5173` 和 `localhost:3000`。检查 `backend/app/main.py` 中的 `allow_origins` 列表。

**Q: 后端启动报 MySQL 连接失败？**
确认 MySQL 已启动，且 `.env` 中的 `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME` 正确。

**Q: 聊天没有回复？**
检查 LM Studio (端口 1234) 是否已启动且加载了模型。确认 `OPENAI_BASE_URL` 和 `MODEL_NAME` 配置正确。

**Q: pip 安装太慢？**
使用清华镜像: `pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt`
